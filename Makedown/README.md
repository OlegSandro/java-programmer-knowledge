# Markdown

Markdown – облегчённый язык разметки. Служит для создания текстовых файлов, в которых помимо текста указывается его оформление. Формат текстового файла, написанный на языке Makedown, является легким для восприятия выходного текста при чтении и удобным для исправления исходного кода при редактировании. Является пригодным для преобразования в другие более сложные текстовые форматы (например, HTML, RTF), но при этом не является их полноценной заменой.

# <a name="gaps"></a> Разрывы

## <a name="newlines"></a> Переносы строк

Для того, чтобы последующий текст начинался с новой строки без оформления абзаца, необходимо вставить 2 (или больше) пробела и нажать Enter:

    Текущая строка__⤶
    Новая строка

В результате получим:

    Текущая строка
    Новая строка

---

## <a name="paragraphs"></a> Абзацы

Для того, чтобы начать следующую строку с нового абзаца, необходимо между последней строкой предыдущего абзаца и первой строкой нового абзаца вставить пустую строку (или пару строк):

    Предыдущий абзац⤶
    ⤶
    Новый абзац

В результате получим:

    Предыдущий абзац
    
    Новый абзац

---

## <a name="separators"></a> Разделители

Для того, чтобы между двумя строками вставить разделитель, необходимо между ними вставить новую строку с тремя (или больше) последовательными символами `*` или `-`, при этом между символами допускаются пробелы (даже в различных количествах). Также, если в случае применения в новой строке символов `*`, вставка сверху и снизу дополнительных пустых строк является больше рекомендацией для задания более читабельного оформления исходного кода, то в случае применения символов `-`, это является обязательным условием для срабатывания!

Примеры различных разделителей:

    Первая строка, после которой должен идти разделитель
    *      *      *
    Вторая строка, которая идёт после разделителя

Или:

    Первая строка, после которой должен идти разделитель
    
    ---
    
    Вторая строка, которая идёт после разделителя

В итоге для двух случаев получим одинаковый выходной результат: 

Первая строка, после которой должен идти разделитель
*      *      *
Вторая строка, которая идёт после разделителя

# <a name="headers"></a> Заголовки

## <a name="h1"></a> Заголовки 1-ого уровня

Для того, чтобы указать, что текст является заголовком 1-ого уровня (самого верхнего уровня), есть 2 способа.

### <a name="h1method1"></a> Способ 1

Перед текстом заголовка поставить символ `#`:

    # Заголовок 1-ого уровня

В результате получим:

# Заголовок 1-ого уровня

### <a name="h1method2"></a> Способ 2

В следующей строке после текста заголовка поставить один или несколько символов `=`:

    Заголовок 1-ого уровня
    ======================

В результате получим:

Заголовок 1-ого уровня
======================

# <a name="design"></a> Оформление текста

## <a name="fontstyles"></a> Выделение текста

### <a name="italic"></a> Шрифт курсивом

Для того, чтобы установить для текста шрифт курсивом, необходимо перед текстом и после него поставить по одному символу `*` или по одному символу `_`. Эти символы не взаимозаменяемы, т.е. если для обозначения начала выделения шрифта курсивом шёл символ `_`, то и для обозначения конца выделения шрифта курсивом должен идти символ `_`. При помощи символа `*` можно задать выделение шрифта курсивом даже для частей слова вплоть до одного символа.

Пример выделения шрифта курсивом:

    Последнее слово в этой строке должно идти *курсивом*

Или:

    Последнее слово в этой строке должно идти _курсивом_

В итоге для двух случаев получим одинаковый выходной результат:

Последнее слово в этой строке должно идти *курсивом*

### <a name="bold"></a> Полужирный шрифт

Для того, чтобы установить для текста полужирный шрифт, необходимо перед текстом и после него поставить по два символа `*` или по два символа `_`. Эти символы не взаимозаменяемы, т.е. если для обозначения начала выделения шрифта полужирным применялся символ `_`, то и для обозначения конца выделения шрифта полужирным должен применяться символ `_`. При помощи символа `*` можно задать выделение шрифта полужирным даже для частей слова вплоть до одного символа.

Пример выделения шрифта полужирным:

    Последнее слово в этой строке должно быть **полужирным**

Или:

    Последнее слово в этой строке должно быть __полужирным__

В итоге для двух случаев получим одинаковый выходной результат:

Последнее слово в этой строке должно быть **полужирным**

### <a name="bolditalic"></a> Полужирный шрифт курсивом

Для того, чтобы установить для текста и шрифт курсивом, и полужирный, необходимо перед текстом и после него поставить по три символа `*` или по три символа `_`. Эти символы не взаимозаменяемы, т.е. если для обозначения начала выделения шрифта полужирным применялся символ `_`, то и для обозначения конца выделения шрифта полужирным должен применяться символ `_`. При помощи символа `*` можно задать выделение шрифта и полужирным, и курсивом даже для частей слова вплоть до одного символа.

Пример выделения шрифта полужирным и курсивом:

    Второе ***слово*** в этой строке должно быть оформлено и полужирным, и курсивом

Или:

    Второе ___слово___ в этой строке должно быть оформлено и полужирным, и курсивом

В итоге для двух случаев получим одинаковый выходной результат:

Второе ***слово*** в этой строке должно быть оформлено и полужирным, и курсивом

---

### <a name="strikethrough"></a> Перечеркнутый шрифт

Для того, чтобы установить для текста перечеркнутый шрифт, необходимо перед текстом и после него поставить по одному символу `~`, при этом таким образом можно задать выделение шрифта перечеркнутым даже для частей слова вплоть до одного символа.

Пример выделения шрифта перечеркнутым:

    Второе ~слово~ в этой строке должно быть перечеркнутым

В результате получим:

Второе ~слово~ в этой строке должно быть перечеркнутым

---

## <a name="specialsymbols"></a> Специальные символы

Для того, чтобы вставить в некоторые участки текста специальные символы так, чтобы они воспринимались в буквальном, а не служебном смысле, необходимо перед специальным символом поставить символ `\`. Таким образом, такие символы, как `\`, ``` ` ```, `*`, `-`, `+`, `_`, `#`, `!`, `(` и `)`, `[` и `]`, `<` и `>` и т.д. в конечном результате будут отображаться так, как они пишутся, а не будут служить для объявления, например, списков, выделения текста, цитат и т.д.:

    Список существующих видов скобок:
    
    - круглые – \( и \)
    - квадратные – \[ и \]
    - угловые – \< и \>
    - фигурные – \{ и \}

В результате получим:

Список существующих видов скобок:

- круглые – \( и \)
- квадратные – \[ и \]
- угловые – \< и \>
- фигурные – \{ и \}

## <a name="extrafeatures"></a> Дополнительные возможности

Так как язык разметки Markdown является совместимым с языком разметки HTML, то он поддерживает вставки кода на языке HTML для того, чтобы добавлять в текст специфические конструкции и возможности, которые по умолчанию не поддерживаются в языке разметки Markdown. Например, таким образом можно вставить в текст таблицу при помощи HTML-тегов:

    <table>
      <tr>
        <td>1</td>
        <td>2</td>
        <td>3</td>
      </tr>
      <tr>
        <td>4</td>
        <td>5</td>
        <td>6</td>
      </tr>
      <tr>
        <td>7</td>
        <td>8</td>
        <td>9</td>
      </tr>
    </table>

В результате получим:

<table>
  <tr>
    <td>1</td>
    <td>2</td>
    <td>3</td>
  </tr>
  <tr>
    <td>4</td>
    <td>5</td>
    <td>6</td>
  </tr>
  <tr>
    <td>7</td>
    <td>8</td>
    <td>9</td>
  </tr>
</table>

# <a name="links"></a> Ссылки

## <a name="automatichyperlinks"></a> Автоматические гиперссылки

### <a name="urlhyperlink"></a> Гиперссылка на URL-адрес

Для того, чтобы вставить в текст гиперссылку на определенный URL-адрес, необходимо просто вставить текст с URL-адресом посреди основного текста и тогда он автоматически преобразуется в гиперссылку или же заключить его в угловые скобки (`<` и `>`):

    Перейдите по адресу https://google.com, чтобы найти что-то в интернете

Или:

    Перейдите по адресу <https://google.com>, чтобы найти что-то в интернете

В итоге для двух случаев получим одинаковый выходной результат:

Перейдите по адресу <https://google.com>, чтобы найти что-то в интернете

---

### <a name="emailhyperlink"></a> Гиперссылка на адрес электронной почты

Для того, чтобы вставить в текст гиперссылку на определенный адрес электронной почты, необходимо просто вставить текст с адресом электронной почты посреди основного текста и тогда он автоматически преобразуется в гиперссылку или же заключить его в угловые скобки (`<` и `>`):

    Отправьте на адрес электронной почты test@example.com результат своего теста

Или:

    Отправьте на адрес электронной почты <test@example.com> результат своего теста

В итоге для двух случаев получим одинаковый выходной результат:

Отправьте на адрес электронной почты <test@example.com> результат своего теста

---

## <a name="namedhyperlinks"></a> Гиперссылки с указанием текста

Для того, чтобы вставить в текст гиперссылку, которая будет отображаться не в виде URL-адреса, а в виде другого кликабельного текста, есть 2 способа.

### <a name="namedhyperlinksmethod1"></a> Способ 1

Заключить текст гиперссылки в квадратные скобки (`[` и `]`), а адрес гиперссылки – в круглые скобки (`(` и `)`). Внутри этих круглых скобок также можно по желанию указать подсказку для гиперссылки, которая будет появляться при наведении курсора на гиперссылку. При этом текст подсказки необходимо заключить в двойные кавычки (`"`) или одинарные кавычки (`'`) или круглые скобки (`(` и `)`).

В качестве адреса гиперссылки можно указывать URL-адрес (например, `https://google.com`) или относительный путь от расположения текущей веб-страницы (например, `../../../`).

Пример гиперссылки, оформленной текущим способом:

    Перейдите в [Google](https://google.com "Самая популярная поисковая система в мире"), чтобы найти что-то в интернете

В результате получим:

Перейдите в [Google](https://google.com "Самая популярная поисковая система в мире"), чтобы найти что-то в интернете

### <a name="namedhyperlinksmethod2"></a> Способ 2

Заключить текст гиперссылки в одни квадратные скобки (`[` и `]`), а метку, которая является идентификатором гиперссылки, – в другие квадратные скобки. После этого метка может быть описана в любой строке исходного кода (даже до строки с использованием текущей гиперссылки), при этом такая строка не выводится в конечном результате. 

Формат описания гиперссылки состоит из последовательно указанных идентификатора гиперссылки, заключенного в квадратные скобки (`[` и `]`), двоеточия, текста с адресом гиперссылки (для лучшей читабельности обычно указывается после пробела) и по желанию текста подсказки, который указывается обязательно после пробела и заключен в двойные кавычки (`"`) или одинарные кавычки (`'`) или круглые скобки (`(` и `)`). 

Метки не являются чувствительными к регистру и могут содержать в своём названии как буквы, так и цифры, специальные символы и даже пробелы.

В качестве адреса гиперссылки можно указывать URL-адрес (например, `https://google.com`) или относительный путь от расположения текущей веб-страницы (например, `../../../`).

Пример гиперссылки, оформленной текущим способом:

    [LINK1]: https://google.com "Самая популярная поисковая система в мире"
    
    Перейдите в [Google][link1], чтобы найти что-то в интернете

В результате получим:

[LINK1]: https://google.com "Самая популярная поисковая система в мире"

Перейдите в [Google][link1], чтобы найти что-то в интернете

Также можно при оформлении гиперссылки таким способом не указывать название метки при использовании гиперссылки, оставив соответствующие квадратные скобки пустыми или вообще опустив их целиком, в таком случае при описании гиперссылки меткой будет служить текст гиперссылки, заключенный в первые квадратные скобки при использовании гиперссылки:

    Перейдите в [Google][], чтобы найти что-то в интернете
    
    [Google]: https://google.com "Самая популярная поисковая система в мире"

В результате получим:

Перейдите в [Google][], чтобы найти что-то в интернете

[Google]: https://google.com "Самая популярная поисковая система в мире"

## <a name="innerlinks"></a> Внутренние ссылки

Так как язык разметки Markdown является совместимым с языком разметки HTML и он поддерживает вставки кода на языке HTML для того, чтобы расширять текст специфическими конструкциями и возможностями языка HTML, то это предоставляет возможность вставки в текст на языке Markdown внутренних ссылок. Это такие ссылки, которые ссылаются на другую часть этого же текста. Для того, чтобы вставить в текст внутреннюю ссылку, необходимо воспользоваться возможностями языка Markdown по вставке в адрес гиперссылки относительного пути от расположения текущей веб-страницы и возможностями языка HTML по объявлению якоря для гиперссылки при помощи атрибута `name` тега `<a>`.

В качестве якоря могут указываться различные буквы, цифры и служебные символы (даже в начале), исключая пробел. При этом буквы в якоре не являются чувствительными к регистру.

Пример внутренней ссылки:

    Нажмите [сюда](#3paragraph), чтобы перейти на третий абзац
    
    Второй абзац
    
    <a name="3PARAgraph"></a> Третий абзац

В результате получим:

Нажмите [сюда](#3paragraph), чтобы перейти на третий абзац

Второй абзац

<a name="3PARAgraph"></a> Третий абзац

# <a name="blocks"></a> Блоки в тексте

## <a name="codelinefragments"></a> Кодовые фрагменты строк

Для того, чтобы указать, что текст является фрагментом строки кода, необходимо перед этим текстом и после него поставить обратный апостроф (``` ` ```). Несмотря на то, что кодовые фрагменты строк похожи на листинги (и там, и там текст не обрабатывается, представлен моноширинным шрифтом и выделяется более темным фоном), имеется и существенное различие между ними – кодовые фрагменты строк в конечном результате располагаются посреди текста в то время, как листинги заставляют располагать своё содержимое принудительно со следующей строки в отдельном абзаце.

Пример кодового фрагмента строк:

    Для примера в программе на Java инициализируем целочисленную переменную `boolean d = 42;` и установим ей в последствии другое значение `d = 1;`

В результате получим:

Для примера в программе на Java инициализируем целочисленную переменную `boolean d = 42;` и установим ей в последствии другое значение `d = 1;`

## <a name="listings"></a> Листинги

Для того, чтобы указать, что текст является исходным кодом программы или разметки, необходимо оформить текст c абзаца, который должен идти с отступа в 4 пробела. Как только встретится строка без отступа, это и будет признаком завершения листинга. Другими словами, в листинг можно заключать даже несколько абзацев, если все они, включая строки между абзацами, будут идти с отступа в 4 пробела. Внутри листинга текст никак не обрабатывается, даже если он оформлен согласно обычному синтаксису Markdown. Каждая новая строка в листинге соответствует каждой новой строке в конечном результате.

Несмотря на то, что листинги похожи на кодовые фрагменты строк (и там, и там текст не обрабатывается, представлен моноширинным шрифтом и выделяется более темным фоном), имеется и существенное различие между ними – кодовые фрагменты строк в конечном результате располагаются посреди текста в то время, как листинги заставляют располагать своё содержимое принудительно со следующей строки в отдельном абзаце.

Пример листинга:

    Это обычный абзац, после которого идёт листинг кода программы:
    
        a = 2;
        b = 1;
        c = a + b;

В результате получим:

Это обычный абзац, после которого идёт листинг кода программы:
    
    a = 2;
    b = 1;
    c = a + b;

## <a name="quotes"></a> Цитаты

Для того, чтобы указать, что текст является цитатой, необходимо его оформить в виде абзаца и поставить символ `>` в начале первой строки абзаца (также допускается ставить символ `>` перед каждой строкой цитаты, но это не значит, что в выходном результате эти строки будут идти с новой строки), таким образом, признак начала нового абзаца будет служить признаком конца цитаты:

    Как говорил Сократ:
    > Я знаю, 
    что ничего не знаю.
    
    Хотя некоторые источники утверждают, что эти слова принадлежат Демокриту.

Или:

    Как говорил Сократ:
    > Я знаю, 
    > что ничего 
    > не знаю.
    
    Хотя некоторые источники утверждают, что эти слова принадлежат Демокриту.

В итоге для двух случаев получим одинаковый выходной результат:

Как говорил Сократ:
> Я знаю, 
> что ничего 
> не знаю.

Хотя некоторые источники утверждают, что эти слова принадлежат Демокриту.

Если необходимо, чтобы новый абзац тоже был частью той же цитаты, то необходимо поставить символ `>` и в начале первой строки предыдущего абзаца, и в начале первой строки нового абзаца, и в строке, которая разделяет эти абзацы:

    >Правильно сформулированный вопрос – половина ответа.
    >
    >Гений видит ответ прежде, чем вопрос.

В результате получим:

>Правильно сформулированный вопрос – половина ответа.
>
>Гений видит ответ прежде, чем вопрос.

Если в цитату нужно поместить вложенную цитату, то в начале текста такой цитаты необходимо поставить символ `>`, а в начале текста вложенной цитаты поставить 2 символа `>`, при этом если после вложенной цитаты встретится признак начала нового абзаца внутри цитаты, то это будет признаком конца вложенной цитаты:

    >Как говорил Сократ:
    >>Я знаю, 
    что ничего не знаю.
    >
    >Хотя некоторые источники утверждают, что эти слова принадлежат Демокриту.

В результате получим:

>Как говорил Сократ:
>>Я знаю, 
что ничего не знаю.
>
>Хотя некоторые источники утверждают, что эти слова принадлежат Демокриту.

А вот если после вложенной цитаты встетится признак начала нового абзаца (относящегося уже не к содержимому цитаты), то это будет признаком конца всей цитаты:

    >Как говорил Сократ:
    >>Я знаю, 
    что ничего не знаю.
    
    >Хотя некоторые источники утверждают, что эти слова принадлежат Демокриту.

В результате получим:

>Как говорил Сократ:
>>Я знаю, 
что ничего не знаю.

>Хотя некоторые источники утверждают, что эти слова принадлежат Демокриту.

Максимальный уровень вложенности цитат – 15-ый.

## <a name="lists"></a> Списки

### <a name="numberedlists"></a> Нумерованные списки

Для того, чтобы создать нумерованный (упорядоченный) список, необходимо, чтобы текст каждого элемента списка начинался с новой строки, в начале которой поставлено любое число (порядок чисел не важен, важно только то, что нумерация списка начнется с того числа, которое указано для первого элемента списка) и символ `.` (или символ `)`, но эти символы не взаимозаменяемы):

    Список:
    1. Элемент списка 1;
    2. Элемент списка 2;
    3. Элемент списка 3;
    3. Элемент списка 4;
    1. Элемент списка 5.

Или:

    Список:
    1) Элемент списка 1;
    4) Элемент списка 2;
    3) Элемент списка 3;
    4) Элемент списка 4;
    5) Элемент списка 5.

В итоге для двух случаев получим одинаковый выходной результат:

Список:
1. Элемент списка 1;
2. Элемент списка 2;
3. Элемент списка 3;
3. Элемент списка 4;
1. Элемент списка 5.

Внутри любого элемента списка можно создавать абзацы, но если признак нового абзаца встретился, а в качестве текста нового абзаца идёт следующий элемент списка, то новый абзац не создастся и нумерация списка не собъется. Для того, чтобы новый абзац был внутри элемента списка, необходимо, чтобы первая строка нового абзаца шла с отступа в 3 пробела относительно начала (позиции перед маркером элемента) того элемента списка, в котором он будет находиться, иначе создастся абзац не внутри текущего элемента списка.

### <a name="bulletedlists"></a> Маркированные списки

Для того, чтобы создать маркированный (ненумерованный, неупорядоченный) список, необходимо, чтобы текст каждого элемента списка начинался с новой строки, в начале которой поставлен символ `*` или символ `-` или символ `+` (перемешивать эти символы в рамках одного списка можно, но не рекомендуется):

    Список:
    * Элемент списка 1;
    * Элемент списка 2;
    * Элемент списка 3;
    * Элемент списка 4;
    * Элемент списка 5.

Или:

    Список:
    - Элемент списка 1;
    - Элемент списка 2;
    - Элемент списка 3;
    - Элемент списка 4;
    - Элемент списка 5.

Или:

    Список:
    + Элемент списка 1;
    + Элемент списка 2;
    + Элемент списка 3;
    + Элемент списка 4;
    + Элемент списка 5.

В итоге для всех трех случаев получим одинаковый выходной результат:

Список:
+ Элемент списка 1;
+ Элемент списка 2;
+ Элемент списка 3;
+ Элемент списка 4;
+ Элемент списка 5.

Внутри любого элемента списка можно создавать абзацы, но если признак нового абзаца встретился, а в качестве текста нового абзаца идёт следующий элемент списка, то новый абзац не создастся, а список продолжится без каких-либо признаков вставки абзаца. Для того, чтобы новый абзац был внутри элемента списка, необходимо, чтобы первая строка нового абзаца шла с отступа в 2 пробела относительно начала (позиции перед маркером элемента) того элемента списка, в котором он будет находиться, иначе создастся абзац не внутри текущего элемента списка.

### <a name="multilevellists"></a> Многоуровневые списки

Для того, чтобы создать многоуровневый список, необходимо, чтобы текст каждого элемента списка начинался с новой строки, в начале которой поставлен соответствующий определенному уровню иерархии признак начала нового элемента списка. Элементы списка 1-ого (верхнего) уровня иерархии указываются так же, как и для нумерованного или маркированного списка. Элементы списка последующих уровней иерархии указываются аналогично, только перед признаком начала нового элемента списка необходимо использовать отступ, соответствующий уровню иерархии. 

#### <a name="multilevelnumberedlists"></a> Многоуровневые нумерованные списки

Для нумерованного списка отступ через 3 пробела обозначает каждый новый уровень иерархии. Таким образом, для 2-ого уровня иерархии нумерованного списка нужно поставить 3 пробела от отступа 1-ого уровня иерархии, для 3-его уровня – 6 пробелов от отступа 1-ого уровня иерархии, для 4-ого уровня – 9 пробелов и т.д. Достаточно удобно к элементам списка разных уровней иерархии применять разные признаки начала нового элемента списка, например, можно элементы 1-ого уровня иерархии списка помечать как число с символом `.`, 2-ого уровня – число с символом `)` и т.д., в таком случае можно будет не только по отступам различить, какой элемент списка к какому уровню иерархии относится, но и по признаку начала нового элемента списка.

Таким образом, для 2-ого уровня иерархии нумерованного списка нужно поставить 3 пробела от отступа 1-ого уровня иерархии, для 3-его уровня – 6 пробелов от отступа 1-ого уровня иерархии, для 4-ого уровня – 9 пробелов и т.д. Достаточно удобно к элементам списка разных уровней иерархии применять разные признаки начала нового элемента списка, как это было описанно выше для нумерованных или маркированных списков, когда можно будет не только по отступам различить, какой элемент списка к какому уровню иерархии относится, но и по признаку начала нового элемента списка.

Пример многоуровневого смешанного списка:

    Список 3:

## <a name="images"></a> Изображения

Для того, чтобы вставить в текст изображение, есть 2 способа.

### <a name="imagesmethod1"></a> Способ 1

После знака восклицания (`!`) заключить альтернативный для изображения текст в квадратные скобки (`[` и `]`), а адрес размещения изображения – в круглые скобки (`(` и `)`). Внутри этих круглых скобок также можно по желанию указать подсказку для изображения, которая будет появляться при наведении курсора на изображение. При этом текст подсказки необходимо заключить в двойные кавычки (`"`) или одинарные кавычки (`'`) или круглые скобки (`(` и `)`).

В качестве адреса расположения изображения можно указывать URL-адрес (например, `https://www.oracle.com/a/ocom/img/cw24-java-info-bkgd-2244997.jpg`) или относительный путь от расположения текущей веб-страницы (например, `../../cw24-java-info-bkgd-2244997.jpg`).

### <a name="imagesmethod2"></a> Способ 2

После знака восклицания (`!`) заключить альтернативный для изображения текст в одни квадратные скобки (`[` и `]`), а метку, которая является идентификатором изображения, – в другие квадратные скобки. После этого метка может быть описана в любой строке исходного кода (даже до строки с использованием текущего изображения), при этом такая строка не выводится в конечном результате. 

Формат описания изображения состоит из последовательно указанных идентификатора изображения, заключенного в квадратные скобки (`[` и `]`), двоеточия, текста с адресом размещения изображениягиперссылки (для лучшей читабельности обычно указывается после пробела) и по желанию текста подсказки, который указывается обязательно после пробела и заключен в двойные кавычки (`"`) или одинарные кавычки (`'`) или круглые скобки (`(` и `)`). 

Метки не являются чувствительными к регистру и могут содержать в своём названии как буквы, так и цифры, специальные символы и даже пробелы.

В качестве адреса расположения изображения можно указывать URL-адрес (например, `https://www.oracle.com/a/ocom/img/cw24-java-info-bkgd-2244997.jpg`) или относительный путь от расположения текущей веб-страницы (например, `../../cw24-java-info-bkgd-2244997.jpg`).
